# Поиск границ и подсчёт объектов на изображении

Программа предназначена для автоматического поиска, подсчёта и анализа однотипных объектов на изображении с использованием методов классического компьютерного зрения на языке Python и библиотеки OpenCV.

Система выполняет:

- поиск границ объектов;
- выделение контуров;
- подсчёт количества объектов;
- измерение параметров объектов;
- классификацию формы (круглые / угловатые);
- сохранение обработанных изображений;
- формирование CSV-отчёта.

---

# Возможности программы

После запуска программа автоматически:

1. Загружает изображения из папки `dataset`;
2. Выполняет предобработку изображений;
3. Применяет алгоритмы поиска границ:
   - Sobel;
   - Laplacian;
   - Canny;
4. Находит контуры объектов;
5. Фильтрует шум по площади;
6. Вычисляет характеристики объектов:
   - площадь;
   - периметр;
   - ограничивающий прямоугольник;
   - описывающую окружность;
   - коэффициент округлости;
7. Разделяет объекты на:
   - круглые;
   - прямоугольные (или угловатые);
8. Нумерует найденные объекты;
9. Сохраняет результат в папку `results`;
10. Создаёт CSV-отчёт с количеством объектов.

# Подготовка изображений

Перед запуском программы **необходимо поместить изображения в папку `dataset`**.

Поддерживаемые форматы:

- `.jpg`
- `.jpeg`
- `.png`

### Пример

```text
dataset/
img1.jpg
img2.png
img3.jpeg
```

## Требования к изображениям

Для корректной работы рекомендуется использовать изображения:

с контрастным фоном;
с одинаковыми объектами;
где объекты не слипаются;
с небольшим количеством теней.

### Хороший пример

- монеты на тёмном столе;
- таблетки на светлом фоне;
- болты на белой поверхности.

### Плохой пример

- объекты касаются друг друга;
- слишком тёмное изображение;
- слабый контраст между объектом и фоном.

# Что делает программа

## 1. Предобработка

Изображение:

- переводится в оттенки серого;
- размывается фильтром Gaussian Blur для уменьшения шумов.

---

## 2. Поиск границ

Используются три алгоритма:

### Sobel

Находит изменение яркости:

- по оси X;
- по оси Y.

### Laplacian

Находит резкие перепады интенсивности изображения.

### Canny

Основной алгоритм поиска границ.

Использует:

- сглаживание;
- вычисление градиентов;
- подавление лишних границ;
- двойную пороговую фильтрацию.

Именно `Canny` используется для дальнейшего поиска объектов.

---

## 3. Поиск контуров

После бинаризации программа находит контуры объектов:

```python
cv2.findContours()
```

Мелкие шумы удаляются фильтрацией по площади.

---

## 4. Анализ объектов

Для каждого объекта вычисляются:

### Площадь

```python
cv2.contourArea()
```

### Периметр

```python
cv2.arcLength()
```

### Ограничивающий прямоугольник

```python
cv2.boundingRect()
```

### Описывающая окружность

```python
cv2.minEnclosingCircle()
```

---

## 5. Классификация формы

Используется коэффициент округлости:

```text
4 × π × S / P²
```

где:

- `S` — площадь;
- `P` — периметр.

### Интерпретация



Меньше 0.8 - Круглый

Больше 0.8 - Угловатый

# Обозначения на результате

После обработки программа рисует:

### Зелёный контур

Круглый объект.

### Красный контур

Прямоугольный или угловатый объект.

### Голубая окружность

Минимальная описывающая окружность.

### Розовая точка

Центр объекта.

### Номер объекта

Порядковый номер.

### Информация сверху

- общее количество объектов;
- количество круглых;
- количество прямоугольных.

# Возможные ошибки

## Ошибка: изображения не найдены

Причина:

Папка `dataset` пустая.

Решение:

Добавьте изображения для обработки.

---

## Ошибка загрузки изображения

Причина:

Повреждённый файл или неподдерживаемый формат.

Решение:

Используйте:

- `.jpg`
- `.jpeg`
- `.png`

---

## Объекты определяются неправильно

Причина:

- плохой контраст;
- шум;
- объекты касаются друг друга.

Решение:

Использовать изображения лучшего качества.
